## 4. 画面詳細設計

### 4.1 トップページ（/）

#### 4.1.1 画面概要

ハロープロジェクトファンポータルの入口となる画面。最新ニュース、注目メンバー、今後のイベントをダイジェストで表示し、各コンテンツへの導線を提供します。

#### 4.1.2 URL

- パス: `/`
- タイトル: `ハロープロジェクト総合ファンポータル | Hello! Project Fan Portal`

#### 4.1.3 レイアウト構成（PC）

```

│         最新ニュース                          │
│  ┌───────┐ ┌───────┐ ┌───────┐            │
│  │ニュース│ │ニュース│ │ニュース│ ←グリッド│

├──────────────────────────────────────────────┤
│              [フッター]                       │
└──────────────────────────────────────────────┘
```

#### 4.1.4 主要コンポーネント

1. **メインビジュアル**
   - 背景: グラデーション（ピンク→ブルー）
   - 高さ: 500px（PC）、300px（SP）
   - キャッチコピー: "ハロープロジェクトのすべてがここに"
   - 検索フォーム配置

2. **最新ニュースセクション**
   - 表示件数: 6件（PC）、3件（SP）
   - レイアウト: 3カラムグリッド（PC）、1カラム（SP）
   - ニュースカード使用

3. **注目メンバーセクション**
   - 表示件数: 6件（PC）、スクロール可能（SP）
   - レイアウト: 水平スクロール
   - メンバーカード使用

4. **今後のイベントセクション**
   - 表示件数: 5件
   - レイアウト: タイムライン形式
   - 日付・時間・会場情報表示

5. **グループ一覧セクション**
   - 表示件数: 8件（現役グループ）
   - レイアウト: 4カラムグリッド（PC）、2カラム（SP）

#### 4.1.5 データ取得API

```typescript
// 最新ニュース
GET /api/news?limit=6&sort=date_desc

// 注目メンバー（最近加入、誕生日近い、など）
GET /api/members/featured?limit=6

// 今後のイベント
GET /api/events?date_from=today&limit=5&sort=date_asc

// 現役グループ
GET /api/groups?status=active&limit=8
```

#### 4.1.6 ユーザーインタラクション

- メインビジュアル検索: 検索キーワード入力 → 検索結果ページへ遷移
- ニュースカードクリック: ニュース詳細ページへ遷移
- メンバーカードクリック: メンバー詳細ページへ遷移
- イベント項目クリック: イベント詳細ページへ遷移
- グループカードクリック: グループ詳細ページへ遷移
- 各セクションの「もっと見る」: 各一覧ページへ遷移

#### 4.1.7 アニメーション仕様

- ページロード時: メインビジュアルフェードイン（800ms）
- スクロール時: 各セクションフェードイン（600ms、順次表示）
- カードホバー時: スケール 1.05倍、影強調（300ms）
- メンバーセクション: 水平スクロール（スムーズスクロール有効）

#### 4.1.8 レスポンシブ対応

**PC（1024px以上）**
- 3カラムグリッド（ニュース）
- 4カラムグリッド（グループ）
- 6件横並び（メンバー）

**タブレット（768px - 1023px）**
- 2カラムグリッド（ニュース、グループ）
- 4件横並び（メンバー）

**SP（320px - 767px）**
- 1カラム（ニュース）
- 2カラム（グループ）
- 水平スクロール（メンバー）
- メインビジュアル高さ: 300px

   - カテゴリドロップダウン（全て、イベント、リリース、メディア出演、その他）
